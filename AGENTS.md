# Trading Agent System — 프로젝트 룰

## 1. 프로젝트 목적

AI 에이전트 2팀이 실시간 국내 주식 시장(코스피/코스닥)을 감시하며
각자의 전략으로 종목을 선정하고 수익률을 비교하는 시스템.
총 운영 기간 1개월. 최종 목적은 실제 투자에 반영할 수 있는
AI 투자 파이프라인을 검증하는 것.

---

## 2. 팀 구성 및 AI 엔진 역할

### 팀 A — 스피드팀 (검증 없음)
- 정보 수집 + 분석: Gemini 2.5 Flash
- 종목 최종 결정: DeepSeek R1
- 처리 방식: 직렬 (Gemini → DeepSeek 순차 처리)
- 검증 단계 없음, 트리거 조건 충족 시 즉시 매수 결정
- 시드: 1,000,000원

### 팀 B — 검증팀 (검증 있음)
- 정보 수집 + 분석: DeepSeek V3
- 종목 결정 + 검증: Gemini 2.5 Pro (결정) + DeepSeek R1 (검증) 병렬 실행
- 처리 방식: 직렬(수집) + 병렬(결정/검증 동시 실행)
- Gemini Pro 매수 + R1 승인 시에만 매수 집행
- Gemini Pro 매수 + R1 거절 시 → 거절 (보수적 원칙)
- 시드: 1,000,000원

---

## 3. 데이터 수집 원칙

### 실시간 감시 (토큰 소모 없음)
- 한국투자증권 API: 장중 실시간 호가, 체결, 수급
- pykrx: 시장 전체 수급, 업종 등락, 거래량
- 운영 시간: 08:00 ~ 애프터마켓 종료

### AI 호출 (토큰 소모)
- 트리거 조건 충족 시에만 호출
- 불필요한 AI 호출 금지 — 비용 최적화 원칙

### 공시/뉴스 수집
- DART API: 공시 수집
- Naver API: 뉴스 수집
- 수집 시 반드시 현재 시각 기준 데이터만 사용

---

## 4. Look-ahead Bias 방지 원칙

- 모든 데이터는 현재 시각 기준으로만 수집
- 미래 데이터가 파이프라인에 유입되는 구조 절대 금지
- AI 프롬프트에 반드시 현재 시각 컨텍스트 명시

---

## 5. 트리거 조건 원칙

- 아래 조건 중 2개 이상 동시 충족 시에만 AI 호출
  - 거래량 전일 동시간대 대비 300% 이상
  - 기관 순매수 당일 전환
  - 외국인 3일 연속 순매수
  - 업종 등락률 상위 3개 진입
  - 52주 신고가 근접 또는 돌파
- 트리거 조건은 triggers/conditions.py 에서만 관리

---

## 6. 종목 선정 원칙

- 신규 진입(첫 매수): 1주당 가격 59,000원 이하 종목만
- 보유 중 추가 매수(물타기/불타기): 가격 제한 없음, AI 자율 판단
- 매수 수량 및 종목 수: AI 자율 결정
- 시드 1,000,000원 기준 운용

### 팀 B 한정 — 아래 항목 발견 시 매수 거절
- 관리종목 / 투자경고 / 투자위험 종목
- 공매도 잔고 급증 종목
- DART 최근 3일 내 악재성 공시 종목

---

## 7. 수수료 기준

- 매수 수수료: 0.015%
- 매도 수수료: 0.015%
- 증권거래세: 0.18% (코스닥 기준)
- 모든 수익률 계산 시 수수료 차감 후 실수익 기준

---

## 8. 리포트 생성 원칙

- 생성 주기: 영업일 5일 기준 1주 (매주 금요일 장 마감 후)
- 저장 위치: /reports/ 폴더
- 파일 형식: .md
- 파일명: week1.md / week2.md / week3.md / final.md
- 내용: 거래내역, 수익률 분석, AI 자기평가, 팀 A vs B 비교
- 최종 리포트: 수익률, 승률, MDD, 평균 보유기간, 최종 승자 판정

---

## 9. 코드 작성 규칙

- 환경변수는 반드시 .env에서만 로드, 코드에 직접 키 입력 금지
- 각 모듈 역할 고정
  - data/market.py: pykrx 시장 데이터만
  - data/realtime.py: 한투 API 실시간 데이터만
  - data/dart.py: DART 공시 수집만
  - triggers/conditions.py: 트리거 조건 로직만
  - agents/team_a.py: 팀 A 에이전트 로직만
  - agents/team_b.py: 팀 B 에이전트 로직만
  - tracker/portfolio.py: 수익률 추적만
  - reports/generator.py: 리포트 생성만
- 모듈 간 역할 침범 금지
- AI 호출 함수는 반드시 try/except 처리
- 모든 AI 호출 로그 반드시 기록 (종목 선정 근거 포함)

## 10. Git 푸시 규칙

### 절대 푸시 금지
- .env 및 모든 환경변수 파일
- venv/ 가상환경 폴더
- __pycache__/ 및 .pyc 파일
- logs/ 로그 파일
- data/cache/, data/temp/ 임시 데이터
- tracker/history/ 수익률 기록 로컬 파일

### 커밋 메시지 규칙
- 기능 추가: feat: 설명
- 버그 수정: fix: 설명
- 설계 변경: refactor: 설명
- 룰/문서 수정: docs: 설명

### 브랜치 규칙
- main: 안정된 버전만
- dev: 개발 중인 버전

## 12. 장기 방향성 원칙

- 이 시스템은 궁극적으로 사용자용 투자 분석 SaaS로 발전한다
- 모든 AI 판단 근거는 항목별 점수 구조로 저장 (텍스트 결론 X)
- 코드/로직 설계 시 항상 전제:
  "이 로직이 나중에 정형화된 UI 테이블로 바로 출력될 수 있는가?"
- AI는 분석+점수 제공까지만, 최종 판단은 사용자 몫
- 법적 리스크 회피 원칙: "추천"이 아닌 "분석 근거 제공" 구조 유지
