# Trading Agent System — 투자 철학 및 운용 방식

## 1. 프로젝트 기본 원칙

- 운용 기간: 1개월 (영업일 기준)
- 시드머니: 팀당 500,000원 (추가 투입 없음)
- 시드 전액 소진 시 해당 팀 운용 종료, 상대 팀은 끝까지 계속 진행
- 신규 진입 조건: 1주당 가격 59,000원 이하 종목만
- 추가 매수(물타기/불타기): 가격 제한 없음, AI 자율 판단
- 매수 수량 및 종목 수: AI 자율 결정
- 동시 보유 최대 종목 수: 팀 A 3종목 / 팀 B 4종목
- 동일 종목 중복 선정 허용 (두 팀이 같은 종목 동시 매수 가능)
- 실행 방식: 수동 테스트 후 안정화 확인 시 자동 전환 (GitHub Actions)
- 사이드카 / 서킷브레이커 발동 시 무시하고 감시 계속, 해제 후 정상 운용

## 2. 매수 대상 종목 기준

### 대상
- 코스피 / 코스닥 일반 보통주만

### 제외 종목 (절대 매수 금지)

#### 종목 유형
- 우선주
- ETF / ETN
- 리츠 (REITs)
- 스팩 (SPAC)
- 외국주식 DR

#### 종목 상태 이상
- 관리종목 지정
- 투자경고 / 투자위험 / 투자주의 지정
- 단기과열 지정
- 불성실공시법인 지정
- 상장폐지 사유 발생

#### 수급 이상
- 공매도 잔고 전일比 50% 이상 급증
- 대주주 지분 매도 공시
- 임원 / 주요 주주 대량 매도

#### 재무 이상
- 감사의견 거절 또는 한정
- 영업손실 3년 연속
- 자본잠식 50% 이상
- 최근 분기 매출 전년比 -30% 이상 급감

#### 시세 이상
- 당일 이미 상한가 도달한 종목
- 갭 상승 10% 이상 종목
- 52주 신저가 근접 종목
- 거래량 없이 가격만 오르는 종목

## 3. 공통 데이터 수집 원칙

두 팀은 동일한 DB에서 같은 데이터를 기반으로 판단을 시작한다.
수익률 차이는 순전히 AI 엔진과 투자 철학의 차이에서 비롯된 것으로 간주한다.

### 수집 데이터
- 한국투자증권 API: 실시간 호가, 체결가, 수급 (외국인/기관 순매수)
- pykrx: 거래량, 업종 등락률, 수급 데이터
- DART API: 공시 수집 및 긍정/부정/중립 분류
- Naver API: 뉴스 수집

### DB 구조 (SQLite)
- 시장 데이터: 수집 시각, 종목코드, 현재가, 거래량, 외국인/기관 순매수, 체결강도
- 공시 데이터: 공시 시각, 종목코드, 제목, 내용 요약, 긍정/부정/중립 분류
- 트리거 발동 기록: 발동 시각, 종목코드, 충족 조건 목록
- AI 판단 기록: 판단 시각, 팀 구분, 종목코드, 매수/매도/관망, 판단 근거, 확신도, 단타/스윙 구분
- 포트폴리오: 팀 구분, 종목코드, 매수가/수량/시각, 매도가/수량/시각, 수수료, 실현 손익
- 잔고 현황: 팀 구분, 시드 잔액, 평가금액, 누적 수익률

## 4. 팀 구성 및 처리 방식

### 팀 A — 스피드팀
- Gemini 2.5 Flash: 정보 수집 + 분석
- DeepSeek R1: 종목 최종 결정
- 처리 방식: 직렬 (Gemini → DeepSeek 순차)
- 검증 단계 없음, 즉시 매수 집행

### 팀 B — 검증팀
- DeepSeek V3: 정보 수집 + 분석
- Gemini 2.5 Pro + DeepSeek R1: 병렬 실행
  - Gemini 2.5 Pro: 종목 결정
  - DeepSeek R1: 반론 검증
- Gemini Pro 매수 + R1 승인 시에만 매수 집행
- Gemini Pro 매수 + R1 거절 시 → 거절 (보수적 원칙)

### 팀 B 교차 검증 목적
- 할루시네이션 방지: Gemini Pro가 존재하지 않는 호재를 만들어내거나 잘못된 데이터를 기반으로 판단할 경우 R1이 반론 검증으로 차단
- 편향 제거: 단일 AI 판단의 편향을 서로 다른 회사(Google vs DeepSeek)의 교차 검증으로 최소화
- 승인 기준: Gemini Pro 매수 + R1 승인 시에만 매수 집행. 둘 중 하나라도 거절 시 관망 처리
- 이 구조는 팀 A(단독 판단)와의 수익률 비교를 통해 교차 검증의 실효성을 검증하는 것이 목적

- 중립: 보수적 운용 (포지션 축소)

## 7. 판단 우선순위

### 팀 A
1순위: 수급 (외국인/기관 순매수 여부)
2순위: 모멘텀 (거래량 급증, 가격 돌파)
3순위: 섹터 (현재 강세 테마 해당 여부)
4순위: 뉴스 (긍정적 촉매 존재 여부)
→ 1+2 충족이면 즉시 진입

### 팀 B
1순위: 리스크 제거 (제2조 매수 제외 기준 전체 체크 — 하나라도 해당 시 즉시 탈락)
2순위: 수급 (외국인/기관 순매수 여부)
3순위: 재무 이상 없는지 확인
4순위: 모멘텀 (거래량, 가격)
5순위: 뉴스
→ 1순위 통과 후 2+3+4 충족 시 검토 → R1 최종 검증

## 8. 자금 흐름 판단 기준

### 시장 전체 (매수 전 필수 체크)
- 매수 우호: 외국인 코스닥 순매수, 기관 프로그램 매수 우위, 고객예탁금 증가
- 매수 비우호: 외국인 코스닥 3일 연속 순매도, 기관 프로그램 매도 우위
- 비우호 환경 시 두 팀 모두 신규 진입 보류

### 종목별 확신도 기준
- HIGH: 외국인 + 기관 동시 순매수, 거래량 전일比 300% 이상, 체결강도 100 이상
- MEDIUM: 외국인 or 기관 둘 중 하나 순매수, 거래량 전일比 150% 이상
- LOW: 조건 1개만 충족 → 진입 재검토

### 확신도별 포지션 사이징
- HIGH: 가용 현금의 40~50% 투입
- MEDIUM: 가용 현금의 20~30% 투입
- LOW: 가용 현금의 10~15% 투입

### 절대 진입 금지 (두 팀 공통)
- 외국인 순매도 5일 연속
- 코스닥 거래대금 전주比 40% 이상 급감
- 반도체/대형주 동반 급락 중

## 9. 단타 / 스윙 구분 기준

### 단타
- 조건: 당일 이슈성 뉴스, 거래량 단기 급등

- 목표: +10~15% / 손절: -5%
- 보유 기간: 3일 ~ 2주

→ AI가 진입 시 단타/스윙 명시 후 해당 기준 자동 적용

## 9-1. 지정가 진입 원칙

- AI는 시장가가 아닌 희망 진입가(지정가)로 매수 주문을 낸다
- AI 출력 JSON에 entry_price 필드를 반드시 포함할 것
- entry_price는 현재가 기준으로 AI가 자율 판단하여 설정 (현재가 이하)
- 당일 장 마감(15:20)까지 미체결 시 자동 취소
- 취소된 종목은 다음날 시장 스캔에서 다시 트리거 조건 충족 시 재발굴
- 억지로 다음날까지 주문 연장 금지 — 하루가 지나면 수급/모멘텀이 바뀌기 때문

- 모멘텀 소멸 + 대안 없음 → 매도

## 14. AI 프롬프트 구조

### 시스템 프롬프트 (고정)
너는 한국 주식 단타/스윙 전문 투자 에이전트다.
현재 시각은 {datetime} 이다.
이 시각 이후의 정보는 존재하지 않는다.
너는 투자 전문가의 관점으로 판단하며
판단 우선순위, 매도 기준, 포지션 사이징을 반드시 준수하라.
매수 대상은 코스피/코스닥 일반 보통주만이며
제외 종목 기준에 해당하는 종목은 절대 선정하지 않는다.

### 유저 프롬프트 (매 호출마다 동적)
현재 시장 데이터:
- 시장 방향성: {우호적/중립/비우호적}
- 수급: {외국인/기관 순매수 상위 종목}
- 거래량 급등: {전일比 300% 이상 종목}
- 섹터 흐름: {업종별 등락률}
- 공시: {오늘 DART 주요 공시}
- 보유 현황: {현재 보유 종목 + 수익률}
- 가용 현금: {현재 현금}

### AI 출력 형식 (JSON 고정)
{
  "action": "매수/매도/관망",
  "ticker": "종목코드",
  "name": "종목명",
  "type": "단타/스윙",
  "reason": "판단 근거",
  "confidence": "HIGH/MEDIUM/LOW",
  "quantity": 수량,
  "target_price": 목표가,
  "stop_loss": 손절가
}

5. 팀 A vs 팀 B 비교: 수익률 격차, 선정 종목 차이, 동일 종목 선정 현황

### 최종 리포트 추가 항목
1. 전체 수익률 최종 집계 (수수료 차감 후)
2. 팀 A vs 팀 B 최종 승자 판정
3. 승률, MDD, 평균 보유기간
4. 동일 종목 중복 선정 분석
5. 어떤 전략/엔진이 한국 단타에 더 적합한가 결론

## 18. 장기 비전 (SaaS)

- 현재: AI 에이전트 성능 검증용 모의투자 시스템
- 궁극적 목표: 사용자가 투자의 눈을 키우는 데이터 분석 지팡이

사용자 입력: 필터 (국장/섹터/가격/투자성향)
AI 출력: 종합점수 + 항목별 분석 테이블
  - 재무건전성
  - 가이던스
  - 공시 정보
  - 수급
  - 모멘텀

AI는 근거와 점수까지만 제공
최종 매수 판단은 철저히 사용자의 몫

## 19. 설명 원칙

- 모든 분석 결과는 투자 비전공자도 이해할 수 있는 언어로 출력
- 점수만 주는 게 아니라 "왜 이 점수인지" 한 줄 근거 필수 동반
- 시장 상황 설명은 전문용어 최소화, 맥락 중심으로 서술
- 나쁜 예: "외국인 순매수 전환, 체결강도 120"
- 좋은 예: "외국인이 오늘 처음 사기 시작했고, 매수세가 매도세보다 강한 상태"
- 이 원칙은 모의투자 단계부터 적용하며 SaaS 전환 시 UI 출력 텍스트로 그대로 활용된다

## 20. 운영 5대 철칙

1. **확률적 관점**: "오를 것이다" 같은 확정적 예측 금지. 반드시 가능성과 확률 기반으로 설명.
2. **사용자 주도권**: AI는 분석과 근거를 제안할 뿐, 최종 매매 결정권은 사용자에게 귀속.
3. **데이터 중립성**: 매수 근거만 제시하지 않는다. 리스크와 반대 논리도 함께 분석하여 편향 제거.
4. **출처 투명성**: AI 판단의 근거가 된 데이터 출처(공시, 뉴스, 수급)를 반드시 명시.
5. **비용 효율성**: 트리거 조건 미충족 시 AI 호출 금지. 캐싱 활용으로 불필요한 API 비용 최소화.
